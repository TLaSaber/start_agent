会话与上下文管理

Session 生命周期 · LangGraph Checkpoint 持久化 · Compact 压缩

1. 核心设计决策：复用 LangGraph Checkpoint

LangGraph 内置的 SqliteSaver / PostgresSaver 天然支持图的快照持久化和恢复。我们把 Session 持久化与 Checkpoint 合一：一个 session 对应一个 thread_id，每次 graph 执行完自动落盘。会话恢复 = 从 checkpoint 重放。

2. Session 数据模型

Session ORM 模型 (仅存元数据)
class Session(Base):
    __tablename__ = "sessions"

    id: str            # UUID, 同时用作 LangGraph thread_id
    user_id: str       # 用户标识
    title: str         # 会话标题 (首条消息截取)
    status: str        # "active" | "closed"
    message_count: int # 消息总数 (冗余字段，方便列表展示)
    created_at: datetime
    updated_at: datetime

    # 完整的消息历史存在 LangGraph checkpoint 里
    # 元数据表只负责列表查询和生命周期管理

4. 上下文压缩 (Compact)

在 Observe 节点中，加载消息历史后检查 token 数（用 tiktoken 估算）。当历史 token 超过阈值（如 80% 上下文窗口），触发压缩：

  1. 取最近 N 轮对话 之前的所有消息，调用 LLM 生成摘要
  2. 摘要作为 SystemMessage 注入上下文开头
  3. 早期消息从上下文中移除，但 Checkpoint 中的全量历史不删
  4. 摘要缓存到 state 中，避免每轮重复压缩
Compact 流程 (Observe 节点内部)
def observe(state):
    messages = state["messages"]  # 来自 checkpoint 的全量历史

    if estimate_tokens(messages) > COMPACT_THRESHOLD:
        boundary = find_compact_boundary(messages, keep_recent=6)
        old_messages = messages[:boundary]
        recent_messages = messages[boundary:]

        # 只在摘要为空时调用 LLM (避免重复压缩)
        if not state.get("compact_summary"):
            summary = llm_compact(old_messages)
            state["compact_summary"] = summary

        # 组装精简上下文: [摘要] + [最近6轮]
        context = [SystemMessage(summary)] + recent_messages
    else:
        context = messages

5. API 设计

会话相关端点
POST   /chat              # 发送消息 (body: {session_id?, user_id, message})
GET    /session/{id}      # 查询会话详情 (metadata + messages)
GET    /session/list      # 用户会话列表 (?user_id=xxx)
POST   /session/{id}/close # 关闭会话
DELETE /session/{id}      # 删除会话 + checkpoint 清理
