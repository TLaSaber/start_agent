1. 核心设计

DeepSeek API 本身就是 OpenAI 兼容协议，后续接任何 OpenAI 兼容的模型提供商只需改配置，不用改代码。我们基于 LangChain 的 BaseChatModel 做一层薄封装，统一异步调用、token 统计、错误转译。

2. ModelProvider 接口

抽象接口
class ModelProvider(ABC):
    """模型提供者抽象"""

    @abstractmethod
    def get_chat_model(self, model_name: str | None = None) -> BaseChatModel:
        """返回 LangChain 兼容的 chat model 实例"""
        ...

    @abstractmethod
    def get_available_models(self) -> list[ModelInfo]:
        """列出可用模型"""
        ...

    @abstractmethod
    def count_tokens(self, text: str, model: str | None = None) -> int:
        """估算 token 数 (用于 compact 判断)"""
        ...

class ModelInfo(BaseModel):
    name: str                        # "deepseek-v3"
    provider: str                    # "deepseek"
    max_tokens: int                  # 上下文窗口大小
    capabilities: list[str]          # ["chat", "function_calling", "json_mode"]

3. DeepSeek + OpenAI 兼容实现

OpenAI 兼容实现 (DeepSeek / OpenAI / 任何兼容服务)
class OpenAICompatProvider(ModelProvider):
    """通过 OpenAI 兼容协议对接任何模型服务"""

    def __init__(self, config: ProviderConfig):
        self.config = config
        # config.api_base → "https://api.deepseek.com/v1"
        # config.api_key  → from env
        self._clients: dict[str, ChatOpenAI] = {}

    def get_chat_model(self, model_name: str | None = None) -> ChatOpenAI:
        name = model_name or self.config.default_model
        if name not in self._clients:
            self._clients[name] = ChatOpenAI(
                model=name,
                api_key=self.config.api_key,
                base_url=self.config.api_base,
                temperature=self.config.temperature,
                max_tokens=self.config.max_output_tokens,
            )
        return self._clients[name]

    def count_tokens(self, text: str, model: str | None = None) -> int:
        # 使用 tiktoken 编码器近似估算
        encoding = tiktoken.encoding_for_model(model or "gpt-4")
        return len(encoding.encode(text))

4. 配置驱动切换

config/model.yaml
providers:
  deepseek:
    type: openai_compat
    api_base: "${DEEPSEEK_API_BASE}"
    api_key: "${DEEPSEEK_API_KEY}"
    default_model: "deepseek-v3"
    available_models:
      - name: "deepseek-v3"
        max_tokens: 65536
        capabilities: [chat, function_calling]
      - name: "deepseek-r1"
        max_tokens: 65536
        capabilities: [chat, function_calling]

5. LangGraph 集成方式

在 Think 节点中，通过 model_provider.get_chat_model() 获取实例，绑定 tools schema 后调用：

Think 节点中调用 LLM
async def think_node(state: AgentState, config: RunnableConfig):
    provider = get_model_provider()  # 从 DI 容器获取
    model = provider.get_chat_model()

    # 绑定 tools (由 ToolRegistry 提供)
    tool_schemas = tool_registry.get_llm_tools()
    model_with_tools = model.bind_tools(tool_schemas)

    # 调用 LLM
    response = await model_with_tools.ainvoke(state["messages"])

    # 解析响应: answer or tool_calls
    return parse_llm_response(response)

6. 防腐层的价值

换模型只需要

  • 在 model.yaml 加一个 provider 配置
  • 改 routing 指向新模型
  • Agent Loop 代码零改动
