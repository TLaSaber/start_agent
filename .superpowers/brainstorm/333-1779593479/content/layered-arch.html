整体分层架构

从 HTTP 请求进入 → Agent Runtime → LangGraph → 底层能力，四层分离

各层职责一句话

接入层 — 只做 HTTP 协议的事：解析请求、校验参数、返回 JSON。不碰 Agent 逻辑。

Agent Runtime — 整个系统的核心。管理 LangGraph StateGraph 的执行，组装上下文，注入记忆，调度工具。

能力层 — 所有外部依赖（模型、工具、记忆存储）通过接口抽象，Agent Runtime 只依赖接口，不依赖具体实现。

基础设施层 — 数据库、文件系统等底层资源。MVP 用 SQLite 打底，接口与 PG 兼容，后续无缝切换。

这个分层的意图

  • Agent Runtime 可以独立测试，不需要真实 HTTP 请求
  • 换模型只需改 ModelProvider 实现，不碰 Agent 逻辑
  • 后续加 WebSocket 流式，只需改接入层，Runtime 不动
