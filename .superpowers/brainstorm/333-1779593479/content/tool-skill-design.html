1. Tool 定义与注册

Tool 基类
class BaseTool(ABC):
    name: str                           # 唯一标识, 如 "read_file"
    description: str                    # LLM 可见的功能描述
    parameters: dict                    # JSON Schema, 传给 LLM 的 function params
    risk_level: Literal["low", "medium", "high", "critical"]

    @abstractmethod
    async def execute(self, **kwargs) -> ToolResult:
        """执行工具，返回统一结果"""
        ...

class ToolResult(BaseModel):
    success: bool
    output: str                         # 给 LLM 看的文本输出
    error: str | None
    metadata: dict                      # 执行耗时、token 消耗等

3. Skill 定义

Skill 本质是 提示词模板 + 工具组合 + 执行约束 的集合。它不直接执行，而是指导 LLM 如何组合使用工具。

Skill 数据结构 & 示例
class SkillDefinition(BaseModel):
    name: str                    # "code-review"
    summary: str                 # 一句话简介 (清单展示用)
    description: str             # 完整提示词 (按需加载)
    tools: list[str]             # 该技能推荐的工具白名单
    constraints: list[str]       # 执行约束
    risk_override: dict[str, RiskLevel] | None  # 可覆盖工具默认风险级别

# 示例：代码审查技能
code_review_skill = SkillDefinition(
    name="code-review",
    summary="对代码变更进行审查，输出问题和改进建议",
    description="""你是代码审查专家。按以下步骤执行：
1. 使用 grep_content 找到变更的函数和类
2. 使用 read_file 读取完整文件上下文
3. 从以下维度审查：安全性、性能、可维护性、错误处理
4. 输出结构化的审查报告，包含严重程度分级""",
    tools=["read_file", "grep_content", "search_file"],
    constraints=["不得修改任何文件", "不得执行代码"],
)

4. 渐进式加载与调度

Agent Loop 中的调度流程
Observe 阶段:
  ┌─────────────────────────────────────────────┐
  │ 1. 注入 Skill 清单 (名称 + 一句话简介)       │
  │    → System Prompt: "可用技能: code-review,   │
  │       data-analyze, doc-generator..."        │
  └─────────────────────────────────────────────┘
                      │
Think 阶段:
  ┌─────────────────────────────────────────────┐
  │ 2. LLM 根据用户任务，决定是否调用技能         │
  │    → 若命中: 输出 skill_activate="code-review" │
  │    → 若未命中: 直接选择原子工具               │
  └─────────────────────────────────────────────┘
                      │
Observe (下一轮):
  ┌─────────────────────────────────────────────┐
  │ 3. 检测到 skill_activate → 加载完整 Skill    │
  │    → 注入完整提示词 + 工具白名单 + 约束        │
  │    → 注入 active_skill 到 state              │
  └─────────────────────────────────────────────┘
                      │
Act 阶段:
  ┌─────────────────────────────────────────────┐
  │ 4. 执行前检查: 工具是否在 Skill 白名单内？    │
  │    → 不在白名单 → 拒绝执行                    │
  │    → 在白名单 → 检查风险级别 → 执行           │
  └─────────────────────────────────────────────┘

5. ToolRegistry 注册发现

自动注册机制
class ToolRegistry:
    """启动时自动扫描注册所有 BaseTool 子类"""

    def __init__(self):
        self._tools: dict[str, BaseTool] = {}

    def discover(self, tool_dir: str = "agent/tools"):
        """扫描 tool_dir 下的所有 .py 文件，自动注册 BaseTool 子类"""
        for cls in walk_subclasses(BaseTool):
            instance = cls()
            self._tools[instance.name] = instance

    def get_llm_tools(self) -> list[dict]:
        """生成 OpenAI 兼容的 tools/functions 参数"""
        return [
            {"type": "function", "function": {
                "name": t.name,
                "description": t.description,
                "parameters": t.parameters,
            }}
            for t in self._tools.values()
        ]

    def execute(self, name: str, **kwargs) -> ToolResult:
        tool = self._tools.get(name)
        if not tool:
            raise ToolNotFoundError(name)
        return await tool.execute(**kwargs)
